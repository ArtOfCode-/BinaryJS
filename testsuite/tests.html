PriorityLevel

The PriorityLevel is an indicator of how critical a failure in the tests covered by this level is.

PriorityLevel BuildFailure - the build should not be passed if these tests fail.
PriorityLevel MayFailWithout - other functions in the source may fail if these methods do not pass.
PriorityLevel High - not a game-breaker but may seriously affect the library. Do not pass the build if previous builds passed these tests.
PriorityLevel Medium - not a game-breaker but may affect functionality. Build can still be passed with a cautionary note.
PriorityLevel Low - there is a possibility that a fail may affect functionality but prioritise others.
